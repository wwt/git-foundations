    Make Local Git Repository Changes With Atomic Commits

    It is possible to commit many, many changes at once although doing so can make it difficult to find individual changes within a commit. Alternatively, atomic commits are those which involve a separate commit for each change. While atomic commits can be more tedious, they provide the benefit of isolating changes to individual commits.

    Create and Commit a New Python Script to the Repository

    1. Let's create a new file, a simple Python script, in our repository. The file will be blank to start, but we will add some code to the file shortly. In one series of commands we will:
    2. Create a Python script.
    3. List the contents of the repository
    4. Review the Git repository status.
    touch my_script.py
ls -l
git status

    1. Notice the following:

    2. The my_script.py file exists and is empty (0 bytes).

    3. The my_script.py file is in the Untracked files section of the git status output.

    4. Issue commands to:

    5. Add the my_script.py file to the Git staging area.
    6. Review the Git repository status.
    git add my_script.py
git status

    1. Notice that the my_script.py is now in the Changes to be committed section.

    2. This indicates my_script.py is now in the Git staging area.

    3. Issue commands to:

    4. Commit the my_script.py file to the local Git repository.
      • The -m flag represents the required message that accompanies the commit.
      • There are other ways to add a message to a commit, although a message is required for each commit.
    5. Review the Git repository status.
    git commit -m "Initial commit of 'my_script.py'"

    1. Notice there are no changes to commit. The README.md file does have changes although isn't yet staged for commit; we will work on that shortly.

    Add Python Code to the my_script.py File

    1. To make changes to the my_script.py file, we will add some executable code. Use the following commands to add some Python code:
    echo '#!/usr/bin/python' >> my_script.py
echo '# This script says hello' >> my_script.py
echo 'name = input("What is your name? ")' >> my_script.py
echo 'print(f"It is nice to meet you, {name})' >> my_script.py

    1. View the changes between the copy of my_script.py in the working directory and the local repository with the following command:
    git diff

    1. View the Git status of your local repository with the following command:
    git status

    1. We now have two changed files to stage and commit to the local repository. Issue commands to:
    2. Move all changed files in the working directory to the staging area.
    3. Commit the staged changes to the repository.
    git add .
git commit -m "Appended line to README.md and added commands to my_script.py"

    1. View the Git status of your local repository with the following command:
    git status

    1. Notice the working branch, branch3 shows no changes to commit

    Push Committed Changes to GitHub

    1. We now have several atomic commits to our local repository to push to our GitHub repository. Initiate the push with the following command:
    git push

    1. Notice that we receive a fatal error when we attempt the git push - GitHub does not have a branch which matches the working branch in our local repository (branch3).
    2. Before we can push changes with a simple git push command, we need to create branch3 in GitHub.
    3. Notice in the output, after the fatal error, Git is friendly enough to display the exact command to both create branch3 in GitHub and then push our atomic commits to GitHub.
    4. Initiate the push again with the following command:
    git push --set-upstream origin branch3

    1. Notice that Git creates a new branch, branch3, in GitHub, and pushes the local repository branch3 commits to the GitHub branch3.
