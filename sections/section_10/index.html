    Review Changes in the GitHub Repository

    Let's take a look at how the changes we pushed from our local Git repository impact the GitHub repository.

    Local Repository Push Review

    1. Return to GitHub and notice a few things about the repository:
    2. There is still just one commit on the main branch; we haven't yet merged the changes in branch3 to the main branch.
    3. The GitHub repository now has three branches, up from two before we pushed branch3 from our local Git repository to GitHub.
    4. There is a highlighted message indicating a recently pushed branch (branch3).

    1. Expand the Branch: main drop menu and notice there three branches, main, branch1, and branch3.
    2. Click on branch3:

    1. From the branch3 view, notice GitHub shows us that branch3 is two commits ahead of the main branch.
    2. These two commits represent the two commits we made to our local repository and then pushed to GitHub.
    3. Click on the 3 commits link to review the individual commits.

    1. The branch3 commits view shows three total commits.
    2. One of the commits is the initial commit that occurred when we created the repository.
    3. We can also see the two atomic commits from our local Git repository along with the messages we added with the git commit -m commands.
    4. Even though the git push command sent all of our local repository to GitHub at once, the individual atomic commits remain separate and reviewable.

    1. Click the git-repo-1 link at the top of the window to return to the repository home page.

    Now we can start the process to merge our local changes to the GitHub repository, from branch3, into the main branch. Click the link below to continue:
